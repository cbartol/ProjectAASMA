User Interface

This window contains:

  • Strategy files manager on the top right,

  • Missions Selector below the Strategy files manager,

  • Pierre's AI Selector below the Missions Selector,

  • Strategy Editor on the right which contains:

    • Standard toolbar and new Nanobots toolbar,

    • Group Actions toolbar and Single Action toolbar,

    • Strategy Treeview,

    • Details panel which contains details of the Selected node in the Strategy Treeview,

    • Errors list window on the bottom right.

Strategy files Manager

On the left, you can manage your strategy files:

Click on the first button to create a new empty strategy.
Double click on an existing strategy file or select it and click on Open button to load the strategy in the editor.
Click on the Save button to save your current strategy. The name of the file is the name of your strategy.
Click on the Delete button to delete the selected strategy file.

To test your strategy, you must also select a mission. You can also choose a spectific Pierre's AI if you want.

Once you have loaded your strategy, you can edit it on the right of the window.

Standard and  new Nanobots toolbars

The first button is the Build button. Click on it to see if there is no error in your strategy. If it contains errors, it will appear oin the Error list window (see below).

The arrow allow you to re-order Single Actions only.

Copy, cut, paste and delete button allow you to Copy, cut, paste and delete the selected node in the Strategy TreeView.

Next, each button allow you to add a new kind of Nanobot in your Strategy. You can have several kinds of each Nanobot but one (and only one) NanoAI and one or zero NanoIPCreator.

Group Actions and Single Actions toolbars

Group Actions:
For each Nanobot of your Strategy, you can define one or more States. A State contains one or more Group, Test and Single Actions. Click on the Add State button to add a new State to your selected Nanobot.

In a State or in a Group Actions, you can add a Group Actions. A Group Actions can contain one or more Group, Test and Single Actions. To add a Group Actions, click on the Add Group Actions button.

In a State or a Group Actions, you can add a Test Actions. A Test Actions contains two Group Actions. The first is executed if the Test result is true, the second is executed if the Test result is false. To add a Test Action, click on the Add Test Actions button.

Single Actions:
Those are the different actions your Nanobots can do. It can MoveTo, CollectFrom, TransferTo, etc. Beware: some actions are not allowed for a kind of Nanobot. For example, if you try to add a MoveTo action to a nanoNeedle, you'll see a Build error in the Errors list window. To add a Single action, click on the wanted action button.

Strategy Treeview

This is the representation of your strategy. The first node allow your to name it and to choose your injection point.
Then, for each Nanobot of your Strategy, you have its different states and for each state, Group, Test and Single Actions.

To add a new Nanobot in your Strategy, click in the New Nanobot toolbar.
To add a new State to a Nanobot, select it and click on the New State in Group Actions toolbar.

To edit a node, select it in the Strategy Treeview.

Error list window

If there is an error in your Strategy, this error appear in this window. To go directly to this error, double click on it.

Details panel

In this part, you can edit the selected node.

Strategy Node

If you select the first node of your strategy, you can define the name of your Strategy and the Injection Point wanted.

The name of your strategy is the name of your file. Change it and click on Apply button. Then save it and it will appear in the Strategy files manager.

You can choose your Injection Point in two way:

  • You define a specific location. Your NanoAI will be injected at the nearest valid point in the tissue. For example if you select point (120, 120) and if this point is a bone area, your Injection Point will be the nearest point not in a bone area.
  • You define a Calculated location. At the beginning of the game, your Strategy will calculte the middle of the Hoshimi Points available in the tissue. It does the same with the AZN Points and the navigation objectives points. Then, it calculate a point depending of the weight of each of those points:

    Point = (HoshimiPoint middle * HP weight + AZN Points * AZNP weight + Objectives Points * OBJP weight) /(HP weight + AZNP weight + OBJP weight)

    Then, you can choose to be injected at the nearest Hoshimi Point, AZN Point or Objective Point of the calculated point or keep this calculated point.
    Again, your Strategy will choose the nearest valid point in the tissue.

Once you define your name and your Injection Point, click on Apply button.

Nanobot Node

When you select a Nanobot node, you can change its kind of Nanobot and its Default State. When your nanobot is created in the tissue, its first actions are the actions in its Default State.

State Node

When you select a State node, you can set its name. You can also define it as the default state (you can verify this by selecting then the Nanobot node).

A State contains Group, Test and Single Actions. When all the actions are executed, you can decide to repeat actions from the begining by selecting Forever radio button. If you want that all your actions are repeated a number of times, define it.

A Nanobot can also change its state by listening to events. For each Nanobot of your Strategy, there is 3 events:

  • Nanobot destroyed,
  • Enemy Detected,
  • Other Player Detected.

If you want that your Nanobot execute some actions when one of those events is fired, check it and select it.

Group Actions Node

A Group Actions can contains Group, Test and Single Actions. You can set its name and define if actions are repeated when all are done. For example if you want to build 4 NanoExplorer, add a Group Actions, add a Build action in it and set its repeat to 4. A Group Actions must contains one or more actions. It can't be empty.

Test Actions

This is the more complicated ;-). A Test action contains two Group Action and a test. At the execution of a Test action, the test is...tested. If the result is true, the first Group Actions is executed, if not, the second is executed.

There is two kind of test:

  • Value test allow you to compare two values. For example, you can test if the Number of NanoExplorer1 equal 0. If true, you can add a Build action.

  • Distance Test allow you to compare the distance between 2 points. Fo example here, I test if I'm near a Pierre's Nanobot of any kind (distance <= 10).

You can define complex test. If you check the Show test in Log, result of the test will appear in the log area in the 2D viewer. This can be usefull in "debug mode".

Single Actions

For each kind of actions, you can define parameters. For example, for a MoveTo action:

You define where you want to go. Here, I choose the nearest Hoshimi Point with no NanoNeedle of my team.

In MoveTo, you can also define the distance from the destination you want to go. It can be usefull for example if you choose to attack a Pierres's Nanobot: select the Nearest Pierre's Nanobot points and a distance of 10 cells. You can stay away from it an attack it safely.

 

For each Single action, you can define more details:

  • Change Internal Name allow you to change its name in the viewer,
  • Change State allow you to ... change its state. So next action to be executed will be the first action of the selected State,
  • Log text allow you to write in the log area in the 2D viewer.

For each kond of action you have different parameters:

  • For CollectFrom, TransferTo and DefendTo, you define the location to collect/transfer/defend, the number of turns you collect/transfer/defend and the distance from the location,
  • For Build, you must define which kind of Nanobot your NanoAI must build. If you have a NanoIPCreator in your tissue, you can also ask that it appear at the NanoIPCreator location,
  • ForceAutoDestruction, OpenIP and CloseIP don't have parameters,
  • StopMoving can be usefull to change State or to avoid going directly to a Pierre's Nanobot.

Test your Strategy

Once your strategy is ready, click on the Build button to be sure there is no error. Then, select your Mission, a Pierre's AI if you want and click on the Test Strategy button.
